# REPL First (Brain Layer)

The REPL is not just a tool.
It is a discipline of thinking.

## Why this belongs in /learn

Product docs explain how REPL access works.
This document explains why a human should think through the REPL first.

## Mental model

The REPL is where:
- assumptions become visible
- abstractions get tested
- fear turns into evidence

## Principle

> Inspect before belief.

## Loop

1. Ask a question
2. Inspect runtime state
3. Form a hypothesis
4. Run a tiny probe
5. Adjust understanding

## Anti-patterns

- guessing from UI alone
- making broad code changes before inspection
- treating REPL as a shortcut for unsafe execution

## Good use

- check counts
- inspect shapes
- confirm state transitions
- validate understanding of a resolver or rule

The REPL is the user's microscope.
