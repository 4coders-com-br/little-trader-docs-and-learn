# Contributing to Little Trader Documentation

Thank you for your interest in improving Little Trader documentation and learning materials!

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally
3. **Create a branch** for your changes: `git checkout -b docs/your-topic`
4. **Make your changes** (see guidelines below)
5. **Test links** to ensure they're correct
6. **Commit with clear messages**
7. **Push to your fork** and **create a Pull Request**

## Guidelines

### Documentation

- Use clear, concise language
- Include code examples where helpful
- Use absolute paths for internal links: `/docs/ARCHITECTURE.md`
- Add table of contents for long documents
- Update the main README if adding new major docs

### Learning Materials

- Keep lessons focused and goal-oriented
- Include practical exercises
- Link to relevant documentation
- Maintain consistent formatting
- Update curriculum index when adding new lessons

### File Naming

- Use UPPERCASE for main docs: `ARCHITECTURE.md`
- Use lowercase for supporting materials: `01-intro.md`
- Use descriptive names that reflect content

### Markdown Format

```markdown
# Main Title
Brief description of the document.

## Section 1
Content here...

### Subsection
More content...

## Related Resources
- [Link to doc](./OTHER_DOC.md)
- [External link](https://example.com)
```

### Linking

- **Internal links**: Use relative paths: `[Architecture](./ARCHITECTURE.md)`
- **External links**: Use full URLs: `[GitHub](https://github.com/4coders-com-br)`

## Code of Conduct

- Be respectful and inclusive
- Focus on improving documentation
- Provide constructive feedback
- Help others improve their contributions

## Review Process

1. **Initial Review**: Maintainers check for clarity and accuracy
2. **Feedback**: We may request changes or clarifications
3. **Approval**: Once approved, your PR will be merged

## Questions?

- Open an issue for discussions
- Comment on existing PRs for feedback
- Email: contato@4coders.com.br
